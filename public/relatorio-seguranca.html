Status do Sistema: Seguro

Camada 1: Proteção de Interface (Frontend)

A primeira linha de defesa atua na interface do usuário (UI). Utilizando o roteamento do Next.js e o hook useAuth(), garantimos que visitantes não autenticados sejam barrados instantaneamente e redirecionados para a tela de login (`/login`).

  • Redirecionamento forçado para usuários sem sessão ativa.
  • Prevenção de vazamento de UI/UX em páginas protegidas.

Camada 2: Autenticação via Supabase Auth

A segunda barreira cuida da criptografia de identidade utilizando JSON Web Tokens (JWT). Todo o fluxo de sessão é validado rigorosamente pelo backend. Os tokens expiram periodicamente e se auto-renovam nos bastidores de forma segura.

  • Geração de JWT codificados e gestão de persistência de sessão.
  • Senhas nunca são trafegadas em texto puro (Zero Trust).

Camada 3: Row Level Security (PostgreSQL)

Esta constitui a **proteção definitiva no nível de hardware do BD**. O recurso RLS (Row Level Security) intercepta cada query que desce do frontend. Ele destrincha o JWT para conferir o auth.uid().

CREATE POLICY "Usuários podem ver suas próprias entregas"
ON entregas FOR SELECT
USING (auth.uid() = user_id);
  • Bloqueia 100% dos acessos forjados através de ferramentas como Postman ou APIs diretas.
  • As chaves públicas anônimas não comprometem a privacidade do dado do cliente.
